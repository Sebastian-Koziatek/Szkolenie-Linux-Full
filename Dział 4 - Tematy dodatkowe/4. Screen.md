# Narzędzie screen w RHEL10

## Czym jest screen?

`screen` to program umożliwiający uruchamianie wielu sesji terminalowych w jednym oknie oraz ich odłączanie i ponowne podłączanie. Dzięki temu można kontynuować pracę na zdalnym serwerze nawet po rozłączeniu SSH.

## Instrukcja instalacji screen w RHEL10 (kompilacja ze źródeł)

Jeśli pakiet screen nie jest dostępny w repozytoriach, wykonaj poniższe kroki:

1. Zainstaluj wget (jeśli nie masz):
   ```bash
   sudo dnf install wget -y
   ```
2. Pobierz źródła programu screen:
   ```bash
   wget https://ftp.gnu.org/gnu/screen/screen-4.9.1.tar.gz
   ```
3. Rozpakuj archiwum:
   ```bash
   tar xzf screen-4.9.1.tar.gz
   cd screen-4.9.1
   ```
4. Zainstaluj narzędzia do kompilacji:
   ```bash
   sudo dnf groupinstall "Development Tools"
   ```
5. Zainstaluj bibliotekę ncurses-devel (wymagana do kompilacji):
   ```bash
   sudo dnf install ncurses-devel
   ```
6. Skonfiguruj, skompiluj i zainstaluj program:
   ```bash
   ./configure
   make
   sudo make install
   ```
7. Uruchom screen:
   ```bash
   screen
   ```

---

> **Wskazówka:**
> Jeśli podczas kompilacji pojawią się błędy, sprawdź czy masz zainstalowane wszystkie wymagane pakiety (Development Tools, ncurses-devel).


## Praktyczny przykład demonstracji

**Przykład:**

1. Uruchom screen na serwerze:
    ```bash
    screen
    ```
2. W nowej sesji uruchom polecenie, które generuje plik logu przez dłuższy czas, np.:
    ```bash
    for i in {1..60}; do echo "Linia $i" >> /tmp/demo_screen.log; sleep 1; done
    ```
3. Odłącz się od sesji (Ctrl+a, potem d).
4. Zamknij terminal lub rozłącz SSH.
5. Po kilku minutach połącz się ponownie z serwerem i sprawdź, czy proces nadal działa:
    ```bash
    tail -f /tmp/demo_screen.log
    screen -ls
    screen -r
    ```
6. Zakończ sesję screen poleceniem `exit`.

**Cel demonstracji:**
Pokazanie, że procesy uruchomione w screen działają niezależnie od połączenia SSH i nie przerywają się po rozłączeniu.
