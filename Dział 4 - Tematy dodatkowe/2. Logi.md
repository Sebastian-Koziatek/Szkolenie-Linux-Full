## Czym są logi w systemie Linux (RHEL10)

Logi to pliki tekstowe, w których system operacyjny oraz aplikacje zapisują informacje o swoim działaniu, błędach, ostrzeżeniach i innych zdarzeniach. Dzięki logom administratorzy mogą monitorować stan systemu, diagnozować problemy oraz analizować bezpieczeństwo.

Najważniejsze cechy logów:
- Zawierają szczegółowe informacje o zdarzeniach w systemie
- Pomagają w rozwiązywaniu problemów
- Umożliwiają audyt i analizę bezpieczeństwa
- Są kluczowe w pracy administratora

### Główne lokalizacje logów w RHEL10
- `/var/log/messages` – ogólne logi systemowe
- `/var/log/secure` – logi związane z bezpieczeństwem (logowania, sudo, ssh)
- `/var/log/maillog` – logi poczty
- `/var/log/cron` – logi zadań crontab
- `/var/log/dmesg` – logi jądra systemu
- `/var/log/httpd/` – logi serwera Apache

## Przykłady użycia logów

1. **Wyświetlenie ostatnich wpisów z logu systemowego:**
   ```bash
   tail /var/log/messages
   ```
2. **Filtrowanie logów po słowie kluczowym (np. 'error'):**
   ```bash
   grep 'error' /var/log/messages
   ```
3. **Monitorowanie logu w czasie rzeczywistym:**
   ```bash
   tail -f /var/log/secure
   ```
4. **Wyświetlenie logów z ostatniego rozruchu systemu:**
   ```bash
   journalctl -b
   ```
5. **Sprawdzenie prób logowania SSH:**
   ```bash
   grep 'sshd' /var/log/secure
   ```

## Zadania


1. Znajdź plik z logami systemowymi i wyświetl jego zawartość.
   - Podpowiedź: `cat /var/log/messages`
2. Wyszukaj w logach systemowych wszystkie linie zawierające słowo "fail".
   - Podpowiedź: `grep 'fail' /var/log/messages`
3. Otwórz log `/var/log/dnf.log` i sprawdź, jakie operacje były wykonywane przez menedżer pakietów DNF. Następnie podłącz się do serwera drugą sesją SSH (np. z innego terminala lub komputera) i wykonaj aktualizację systemu poleceniem:
   - `sudo dnf update`
   Po zakończeniu aktualizacji ponownie sprawdź log `/var/log/dnf.log` i znajdź wpisy dotyczące przeprowadzonej aktualizacji. Zwróć uwagę, jak logi rejestrują operacje z różnych sesji.


> **Wskazówka:**
> Logi są podstawowym narzędziem diagnostycznym w pracy administratora. Warto regularnie je analizować i automatyzować ich przeszukiwanie.
